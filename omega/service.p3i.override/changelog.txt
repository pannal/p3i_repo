v0.1.8
- Docs: the L5 override now also flows to the TV (not just Kodi-side
  overlay calc). Requires matching xbmc + amdolby_vision kernel patches
  on aml-4.9-21.3_dev / amlogic-4.9-20:
    - xbmc_force_l5_override sysfs flag in amdolby_vision (kernel)
    - aml_dv_apply_l5_override_sysfs() in AMLUtils (Kodi)
  Without those kernel/Kodi patches the override is Kodi-side only as
  before — the addon writes the setting regardless, so no version-skew
  issues, you just don't get the TV-side effect on older builds.
- Docs: "0,0,0,0" is now a valid override meaning "treat the stream as
  having no bars / full active frame" — for streams whose RPU falsely
  claims letterbox. Previously misclassified as "inactive". Empty value
  is the way to leave the stream's L5 alone.
- New SETTINGS.md example for the "force no bars" case.

v0.1.7
- Docs: cmv40.append entry now spells out the three visible-dependencies
  that gate JSON-RPC writes (DV type = Display-LED, no VP override,
  mode != 2). Earlier docs implied "just works"; in reality it works
  only when those conditions are met, and the addon will log an
  InvalidParams rejection on other DV paths. Same in SETTINGS.md table.
  No code change — Kodi C++ side already has the dependencies wired
  correctly (this is just user-facing documentation catching up).

v0.1.6
- Log the actual JSON-RPC response when Settings.SetSettingValue fails,
  so rejected writes can be diagnosed instead of silently failing with
  just "failed to set".

v0.1.5
- Stop deferring to PlexMod4Kodi. The SB helper defers because PM4K
  manages the same LAV SB setting for Plex playback; this addon has no
  shared-setting contention with PM4K. Plex http(s) streams were already
  skipped by the remote-scheme filter, so the deferral only ever blocked
  override.ini from applying to local files played through PM4K — which
  is the case where the user clearly wants it to apply.

v0.1.3
- Icon: replace the SB-helper-cloned placeholder with a distinct gear

v0.1.2
- Verified-settings table: add all subtitle settings. Two new statuses
  document timing tiers Kodi's subtitle pipeline actually uses:
    live-on-next-cue: cached in m_overlayStyle, refreshed by the
        SubtitlesSettings observer so the next drawn subtitle picks the
        change up (covers fontname, colorpick, bordersize, shadow*, bg*,
        marginvertical, align, override*). Applies within one cue.
    needs-restart: sampled at stream/codec open and never re-read; the
        addon writes anyway but logs a warning that the change won't
        affect the current playback (covers pgshdrtosdr enable flag,
        charset, parsecaptions, locale.subtitlelanguage).
  PGS HDR-to-SDR shader params (brightness/saturation/tonemap/mode) are
  fully live — useful for per-folder tuning of HDR subs.

v0.1.1
- Walk one parent folder so a library-wide override.ini doesn't need to
  be duplicated per subfolder. Standard layouts: episode reaches the
  show folder, movie in per-title subfolder reaches the library root.
  Broadest folder applies first, narrower folder overrides per key —
  same model as INI sections inside a single file.

v0.1.0
- Initial release.
- Per-folder Kodi setting overrides via override.ini in the playing file's
  folder. Format: optional [global] section + [filename or glob] sections;
  later/narrower sections override broader ones for the same key.
- Save/restore lifecycle: saved at AV-start, pushed back on stop/end/error.
- Defers to PlexMod4Kodi (script.plexmod) when it is the active session.
- Skips coreelec.amlogic.dolbyvision.audio.seamlessbranch when
  service.p3i.sb is installed (SB helper owns that key).
- Verified-settings table documents which settings are known to take effect
  mid-playback; warnings emitted for everything outside the table.
